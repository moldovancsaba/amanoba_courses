**Question:** Why start with only a few core events? in a small commercial AI app scenario

A) It saves time and reduces noise
B) It improves UI design in a small commercial AI app context
C) It replaces testing when validating a paid AI workflow
D) It increases token usage for a first release with limited time

**Why correct:** Fewer events are easier to manage and review.
**Why others are wrong:**
- A: This is correct.

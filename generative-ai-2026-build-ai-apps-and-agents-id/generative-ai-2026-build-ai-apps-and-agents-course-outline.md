# Course Outline

Course: Generative AI 2026: Build AI Apps and Agents
Audience: Digital nomads using AI daily who want to build a commercial AI app
Time budget: 15 to 30 minutes per day
Scope: Global, cross industry
Tools: OpenAI, GitHub, Vercel, Stripe

## Lessons 1 to 30

1. Orientation and outcomes
   - Goal: Define a clear product promise and success metrics
   - Deliverable: One page product promise and success checklist

2. Problem selection and niche
   - Goal: Choose a real problem with paying potential
   - Deliverable: Problem shortlist with buyer signals

3. Target user and JTBD
   - Goal: Describe primary users and their core job to be done
   - Deliverable: Persona plus JTBD statement

4. Value proposition and pricing hypothesis
   - Goal: Draft a value proposition and a first pricing idea
   - Deliverable: Value prop and pricing draft

5. Feature scoping for a first release
   - Goal: Define a minimal but sellable feature set
   - Deliverable: MVP scope list with must have and later

6. UX flow and user story map
   - Goal: Sketch the end to end flow
   - Deliverable: User story map and flow steps

7. Tech stack decision
   - Goal: Confirm stack and constraints
   - Deliverable: Stack decision doc

8. Repo setup and Git hygiene
   - Goal: Create a clean GitHub repo and workflow
   - Deliverable: Repo with README, branch rules, and issues

9. Project structure and environment config
   - Goal: Set up folders, config, and env secrets
   - Deliverable: Project skeleton with env template

10. OpenAI API basics and safe patterns
    - Goal: Implement a simple API call and guardrails
    - Deliverable: Working OpenAI call with error handling

11. Prompt design for reliability
    - Goal: Create a stable prompt and test cases
    - Deliverable: Prompt spec and test set

12. Tool calling and actions
    - Goal: Connect an agent to tools or functions
    - Deliverable: Agent tool schema and example call

13. Agent workflow design
    - Goal: Define multi step agent logic
    - Deliverable: Agent workflow diagram and steps

14. Data handling and memory
    - Goal: Decide what to store and how
    - Deliverable: Data retention and storage plan

15. Cost control and rate limits
    - Goal: Add safeguards for spend and abuse
    - Deliverable: Cost guard checklist and limits

16. UI integration and UX copy
    - Goal: Connect the UI to the agent and refine copy
    - Deliverable: UI wired to API with onboarding copy

17. Testing the agent with scenarios
    - Goal: Validate behavior with real cases
    - Deliverable: Scenario test report

18. Error handling and fallbacks
    - Goal: Implement graceful failure paths
    - Deliverable: Fallback logic and error responses

19. Logging and basic analytics
    - Goal: Track usage and outcomes
    - Deliverable: Logging plan and basic event capture

20. Documentation basics
    - Goal: Write setup and usage docs
    - Deliverable: User guide and dev setup doc

21. Security and privacy essentials
    - Goal: Add minimal security and privacy practices
    - Deliverable: Security checklist and privacy notes

22. Vercel deployment and env vars
    - Goal: Deploy to Vercel with staging
    - Deliverable: Live staging deployment

23. Domain and production readiness
    - Goal: Prepare for production release
    - Deliverable: Production checklist

24. Stripe checkout integration
    - Goal: Add Stripe checkout flow
    - Deliverable: Working checkout session

25. Stripe webhooks and access control
    - Goal: Gate paid features based on payment
    - Deliverable: Webhook handler and access gate

26. Pricing page and onboarding
    - Goal: Build a simple sales page and onboarding flow
    - Deliverable: Pricing page and onboarding steps

27. Launch checklist
    - Goal: Prepare a release plan and go live
    - Deliverable: Launch checklist

28. Post launch feedback loop
    - Goal: Collect feedback and prioritize fixes
    - Deliverable: Feedback plan and issue list

29. Growth experiments
    - Goal: Run small experiments to find traction
    - Deliverable: Growth experiment plan

30. Maintenance and iteration plan
    - Goal: Plan the next 30 days of improvements
    - Deliverable: Iteration roadmap

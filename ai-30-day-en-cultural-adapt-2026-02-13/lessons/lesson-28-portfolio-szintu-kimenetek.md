# Lesson 28: Portfolio-level outputs

**In one sentence:** We are now creating a portfolio-worthy package from your best materials.
**Time:** 20-30 minutes
**Tangible output:** 3-element portfolio package

## Learning goal

Support for changing jobs or acquiring clients. In this situation, you are a professional starting your career and you have to make a quick but reliable decision.
By the end of today's practical task, you will have a 3-element portfolio package, which you can use again tomorrow.
Key insight: portfolio-level outputs start to work when you make the output measurable at the start. At the end of the lesson, you will be able to: **turn course outputs into presentable material**

### Success criteria (observable)
- [ ] The specific daily output is ready: **3-element portfolio package**.
- [ ] At least one measurable indicator is attached to the output: **Each of the 3 portfolio elements contains a measurable impact**.
- [ ] There is a separate control step for managing the main risk: **unprovable result**.

### Output you will create today
- **Name:** 3-element portfolio package
- **Format:** editable board and short decision note
- **Where saved:** `ai-rutin/nap-28-portfolio-szintu-kimenetek.md`

## Who

**Primary role:** entry-level professional
**Secondary role:** Mentor
**Involved decision maker:** interviewer

## What

### What is this?

This lesson shows in practice how to apply the **portfolio level outputs** method in a real work situation.
The goal is not a theory, but an output that you can really use in your next working day.

### What isn't this?

This is not a generic AI demo, and it is not busywork.

### 2-minute theory

- In the portfolio, it is not the quantity that matters, but the demonstrable result.
- In real work, a quick fix is ​​more important than a perfect first round.
- Quality comes from ensuring that a 3-element portfolio package is verifiable and repeatable.

### Key concepts

- **Working framework:** a short guide that describes the three objectives, format and limits.
- **Quality gate:** quick check before accepting the output.

## Where

### It works fine here

- changing jobs or supporting customer acquisition in situations where a quick and clear decision is required
- when a 3-element portfolio package gives several people a clear next step

### Do not use this when

- if you cannot prove the result, you cannot cover the risk with human control

### Connection points

- daily handovers between a professional starting a career and a mentor
- short statuses and decision preparation notes
- repetitive tasks, where each of the 3 portfolio elements has a measurable impact

## When

### Use it when

- you are faced with the task of changing jobs or supporting client acquisition, and you have little time
- Based on a 3-element portfolio package, several actors must act in the same way

### Frequency

At least once per workday, and every time for recurring tasks.

### Late warning signs

- 3-element portfolio package needs to be rewritten every time
- you often get back that the content does not address the problem of unprovable results

## Why it matters

### Practical benefits

- a 3-element portfolio package is prepared faster, with fewer rounds
- quality will be more measurable, each of the 3 portfolio elements contains a measurable impact with an indicator
- rework is reduced, because you filter out unprovable result errors early on

### What happens if you skip it

- the number of misunderstandings increases in the situation of job change or customer acquisition support
- more difficult to maintain 3 portfolio elements each contain a measurable impact goal

### Realistic expectations

- It will improve: the practical application of portfolio-level outputs and the quality of the transferable output.
- Not guaranteed: infallible response without human control in every run.

## How

### Step by step

1. Name the work situation and the expected result exactly. 2. Write down the mandatory limits (time frame, scope, style, prohibitions). 3. Request the first AI version in a specific format. 4. Run a 3-point quality gate, paying particular attention to **unprovable result** risk. 5. Fix the output, then save it as a usable template.

### Do / Don't

**Do**
- Enter a short, specific task description so that the format of the 3-element portfolio package is clear from the start.
- Before finalizing, check whether 3 portfolio elements have been met, each of them contains a measurable impact measurement point.


**Don't**
- Do not leave the subject of an unprovable result in the background, even if the answer seems good at first.
- Do not ask for a general text if you need material for a specific decision, job change or customer acquisition support task.

### Common errors and their correction

- **Error:** too general a request for job change or customer acquisition support situation. **Correction:** name the person concerned, the deadline and the decision target.
- **Error:** there is no connection between the output and the measurement of each of the 3 portfolio elements containing a measurable effect. **Fix:** record 1 binary pass/fail result at the end of each run.

### It's ready if

- [ ] **3-element portfolio package** output is ready and saved.
- [ ] You checked the output according to at least 3 quality criteria.
- [ ] You have documented the indicator **Each of the 3 portfolio elements has a measurable impact**.

## Guided exercise

### Inputs

- a short description of a real task about the following situation: **Supporting job change or customer acquisition**
- a short sample of your own, which shows what you consider a good portfolio-level output
- checklist to exclude the risk of unprovable results

### Steps

1. Write down the situation, the goal and how you will measure it in 4 lines. Each of the 3 portfolio elements contains a measurable effect and result.
2. Prepare a sample request for the production of a 3-element portfolio package, then request two different versions.
3. Compare the versions in the table and mark which one can go on for real use.

### Expected output format

| Element | What do you type | Expected result |
| --- | --- | --- |
| Situation | Support for job change or client acquisition | Everyone involved understands from one reading |
| Request frame | Target + context + format + constraint | Minimum of 2 usable versions |
| Quality gate | Accuracy + Risk + Next Step | Clear repair decision |
| Final output | 3-element portfolio package | Saved, reusable template |

> **Common mistake:** Many people only look at the risk of an unprovable result at the end, even though it should already be indicated in the task description.

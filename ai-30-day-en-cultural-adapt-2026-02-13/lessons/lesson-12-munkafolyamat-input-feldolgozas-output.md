# Lesson 12: Work flow: input → processing → output

**In one sentence:** The workflow is stable if you manage input, processing and output separately.
**Time:** 20-30 minutes
**Tangible output:** 3-step workflow plan sheet

## Learning goal

Daily qualification of incoming leads. In this situation, you are the CRM owner, and you need to make a fast but reliable decision.
By the end of today's practical task, you will have a 3-step workflow plan.
Key insight: the input-processing-output model starts to work when you make the output measurable right from the start.

At the end of the lesson, you will be able to: **turn a repetitive task into a reproducible process**

### Success criteria (observable)
- [ ] The specific daily output is ready: **3-step workflow plan sheet**.
- [ ] At least one measurable indicator is attached to the output: **clear input and output for each step**.
- [ ] There is a separate control step for managing the main risk: **ad hoc decisions**.

### Output you will create today
- **Name:** 3-step workflow plan sheet
- **Format:** editable board and short decision note
- **Where saved:** `ai-routine/day-12-workflow-input-processing-output.md`

## Who

**Primary role:** Responsible for CRM
**Secondary role:** marketing automation associate
**Involved decision maker:** marketing director

## What

### What is this?

This lesson shows in practice how to apply the **input-processing-output model** method in a real work situation.
The goal is not a theory, but an output that you can really use in your next working day.

### What isn't this?

This is not a generic AI demo, and it is not busywork.

### 2-minute theory

- The process can be scaled if each step can be checked and handed over to others.
- In real work, a quick fix is ​​more important than a perfect first round.
- The quality here is provided by the fact that the 3-step workflow plan sheet is verifiable and repeatable.

### Key concepts

- **Working framework:** a short guide that describes the three objectives, format and limits.
- **Quality gate:** quick check before accepting the output.

## Where

### It works fine here

- daily qualification of incoming leads in situations where a quick and clear decision is required
- when a 3-step workflow plan sheet gives clear next steps to several people

### Do not use this when

- if you make ad hoc decisions, you cannot cover the risk with human control

### Connection points

- Daily handovers between CRM manager and marketing automation employee
- short statuses and decision preparation notes
- repetitive tasks, where each step has a clear input and output

## When

### Use it when

- you encounter the task of daily qualification of incoming leads, and you have little time
- Based on a 3-step workflow plan, several actors must act in the same way

### Frequency

At least once per workday, and every time for recurring tasks.

### Late warning signs

- 3-step workflow design sheet needs to be rewritten every time
- you often get back that the content does not handle the problem of ad hoc decisions

## Why it matters

### Practical benefits

- a 3-step workflow design sheet is prepared faster, with fewer rounds
- quality will be more measurable with clear input and output indicators for each step
- rework is reduced, because you filter out errors in ad hoc decisions early on

### What happens if you skip it

- the number of misunderstandings increases in the situation of daily qualification of incoming leads
- it is more difficult to keep a clear input and output goal for each step

### Realistic expectations

- It will improve: the practical application of the input-processing-output model and the quality of the deliverable output.
- Not guaranteed: infallible response without human control in every run.

## How

### Step by step

1. Name the work situation and the expected result exactly. 2. Write down the mandatory limits (time frame, scope, style, prohibitions). 3. Request the first AI version in a specific format. 4. Run a 3-point quality gate, paying particular attention to the risk of **ad hoc decisions**. 5. Fix the output, then save it as a usable template.

### Do / Don't

**Do**
- Enter a short, specific task description so that the format of the 3-step workflow plan sheet is clear from the start.
- Before finalizing, check whether a clear input and output measurement point has been met for each step.


**Don't**
- Don't leave the topic of ad hoc decisions in the background, even if the answer seems good at first.
- Do not ask for a general text if you need material for a specific decision for the task of daily qualification of incoming leads.

### Common errors and their correction

- **Error:** too general a request for daily qualification of incoming leads. **Correction:** name the person concerned, the deadline and the decision target.
- **Error:** there is no connection between the output and a clear input and output measurement for each step. **Fix:** record 1 binary pass/fail result at the end of each run.

### It's ready if

- [ ] You have completed and saved the **3-step workflow design sheet** output.
- [ ] You checked the output according to at least 3 quality criteria.
- [ ] You have documented the **clear input and output** indicator for each step.

## Guided exercise

### Inputs

- a short description of a real task about the following situation: **Daily qualification of incoming leads**
- a short own sample that shows what you consider a good input-processing-output model output
- checklist to exclude the risk of ad hoc decisions

### Steps

1. Write in 4 lines the situation, the goal and how you will measure clear input and output results for each step.
2. Prepare a sample request for the production of a 3-step workflow plan sheet, then request two different versions.
3. Compare the versions in the table and mark which one can go on for real use.

### Expected output format

| Element | What do you type | Expected result |
| --- | --- | --- |
| Situation | Daily qualification of incoming leads | Everyone involved understands from one reading |
| Request frame | Target + context + format + constraint | Minimum of 2 usable versions |
| Quality gate | Accuracy + Risk + Next Step | Clear repair decision |
| Final output | 3-step workflow design sheet | Saved, reusable template |

> **Common mistake:** Many people only look at the risk of ad hoc decisions at the end, even though it should already be indicated in the task description.

# Lesson 4: Style and voice - teach him to "write like you" **In a sentence:** The tone of voice holds the brand image together, so this must also be taught to the AI.
**Time:** 20-30 minutes
**Tangible output:** personal style brief and blacklist

## Learning goal

Same message to manager and customer in different tone. In this situation, you are a freelance copywriter and you need to make a quick but reliable decision.
At the end of today's practical task, you will have a personal style task description and prohibition list, which you can use again tomorrow.
Key insight: setting the style and sound starts working when you make the output measurable at the start. At the end of the lesson, you will be able to: **create texts adjusted to your own tone**

### Success criteria (observable)
- [ ] The specific daily output is ready: **personal style task description and ban list**.
- [ ] At least one measurable indicator is attached to the output: **2 versions with suitable tone for 2 target groups**.
- [ ] There is a separate control step to deal with the main risk: **communication not suitable for the target audience**.

### Output you will create today
- **Name:** personal style task description and ban list
- **Format:** editable board and short decision note
- **Where saved:** `ai-routine/day-04-stylus-and-sound-teach-me-to-write-like-you.md`

## Who

**Primary role:** Freelance copywriter
**Secondary Role:** Brand Manager
**Involved decision maker:** customer relationship manager

## What

### What is this?

This lesson shows in practice how to apply the **style and voice adjustment** method in a real work situation.
The goal is not a theory, but an output that you can really use in your next working day.

### What isn't this?

This is not a generic AI demo, and it is not busywork.

### 2-minute theory

- The style can be reproduced if you provide a sample, rule and counterexample.
- In real work, a quick fix is ​​more important than a perfect first round.
- The quality here is provided by the fact that the personal style task description and ban list is verifiable and repeatable.

### Key concepts

- **Working framework:** a short guide that describes the three objectives, format and limits.
- **Quality gate:** quick check before accepting the output.

## Where

### It works fine here

- the same message to the manager and the customer in a different tone of voice in situations where a quick and clear decision is required
- when a personal style task description and blacklist give several people a clear next step

### Do not use this when

- if communication is not suitable for the target audience, you cannot cover the risk with human control

### Connection points

- daily handovers between freelance copywriter and brand manager
- short statuses and decision preparation notes
- repetitive tasks, where 2 versions with the appropriate tone count for 2 target groups

## When

### Use it when

- the same message to the manager and the customer, in a different tone, you encounter a task, and time is short
- based on the personal style task description and ban list, several actors must act in the same way

### Frequency

At least once per workday, and every time for recurring tasks.

### Late warning signs

- personal style task description and prohibition list need to be rewritten every time
- you often get back that the content does not deal with the problem of communication that is not suitable for the target audience

## Why it matters

### Practical benefits

- a personal style task description and ban list are prepared faster, with fewer rounds
- the quality will be more measurable for 2 target groups with indicators of 2 versions with appropriate tone
- rework is reduced, because you filter out communication errors that are not suitable for the target audience early on

### What happens if you skip it

- the number of misunderstandings increases when the same message is sent to the manager and the customer in a different tone
- it is more difficult to keep 2 targets for 2 target groups with a suitable tone of voice

### Realistic expectations

- It will improve: the practical application of style and tone settings and the quality of the transferable output.
- Not guaranteed: infallible response without human control in every run.

## How

### Step by step

1. Name the work situation and the expected result exactly. 2. Write down the mandatory limits (time frame, scope, style, prohibitions). 3. Request the first AI version in a specific format. 4. Run a 3-point quality gate, paying particular attention to the risk of **communication not suitable for the target audience**. 5. Fix the output, then save it as a usable template.

### Do / Don't

**Do**
- Enter a short, specific task description so that your personal style task description and blacklist format are clear from the start.
- Before finalizing, check whether 2 measurement points for 2 versions with appropriate tone have been met for 2 target groups.


**Don't**
- Do not leave the topic of communication that does not suit the target audience in the background, even if the answer seems good at first.
- Don't ask for a general text if you need a specific decision, the same message for the manager and the client in a different tone for the task.

### Common errors and their correction

- **Error:** too general a request is sent for the same message to the manager and customer in a different tone for a situation. **Correction:** name the person concerned, the deadline and the decision target.
- **Error:** there is no connection between the output and the measurement of 2 versions with the appropriate tone for 2 target groups. **Fix:** record 1 binary pass/fail result at the end of each run.

### It's ready if

- [ ] The **personal style task description and ban list** output has been completed and saved.
- [ ] You checked the output according to at least 3 quality criteria.
- [ ] You have documented pointers to **2 versions with appropriate tone for 2 target groups**.

## Guided exercise

### Inputs

- a short description of a real task about the following situation: **Same message to manager and customer in different tone**
- a short own sample that shows what you consider a good style and sound setting as an output
- checklist to exclude the risk of communication not suitable for the target audience

### Steps

1. In 4 lines, write down the situation, the goal and how you will measure the results of 2 versions with a suitable tone for 2 target groups.
2. Prepare a request sample for the production of a personal style task description and blacklist, then request two different versions.
3. Compare the versions in the table and mark which one can go on for real use.

### Expected output format

| Element | What do you type | Expected result |
| --- | --- | --- |
| Situation | Same message to manager and customer in different tone | Everyone involved understands from one reading |
| Request frame | Target + context + format + constraint | Minimum of 2 usable versions |
| Quality gate | Accuracy + Risk + Next Step | Clear repair decision |
| Final output | personal style task description and ban list | Saved, reusable template |

> **Common mistake:** Many people only look at the risk of communication that is not suitable for the target audience at the end, even though it should already be indicated in the task description.

# Lesson 18: Role-matched template package II.

**In one sentence:** Now we build the analysis and decision preparation templates.
**Time:** 20-30 minutes
**Tangible output:** 3 report format templates

## Learning goal

Preparation of a weekly report from an operational and management perspective. In this situation, you are a financial analyst and you have to make a quick but reliable decision.
By the end of today's practical task, you will have 3 report format templates, which you can use again tomorrow.
Key insight: report and analysis templates start working when you make the output measurable right from the start.

At the end of the lesson, you will be able to: **create different reports for different management levels**

### Success criteria (observable)
- [ ] The specific daily output is ready: **3 report format templates**.
- [ ] At least one measurable indicator is attached to the output: **3 different usable reports for 3 target groups**.
- [ ] There is a separate control step for handling the main risk: **inadequate detail**.

### Output you will create today
- **Name:** 3 report format templates
- **Format:** editable board and short decision note
- **Where saved:** `ai-routine/day-18-role-matched-template-package-ii.md`

## Who

**Primary Role:** Financial Analyst
**Secondary role:** operations manager
**Involved decision maker:** CEO

## What

### What is this?

This lesson shows in practice how to apply the **report and analysis templates** method in a real work situation.
The goal is not a theory, but an output that you can really use in your next working day.

### What isn't this?

This is not a generic AI demo, and it is not busywork.

### 2-minute theory

- A good report is tailored to the recipient. The same data requires a different view.
- Fewer but more precise instructions typically give better results.
- Quality comes from ensuring that 3 report format templates is verifiable and repeatable.

### Key concepts

- **Working framework:** a short guide that describes the three objectives, format and limits.
- **Quality gate:** quick check before accepting the output.

## Where

### It works fine here

- preparation of weekly reports in operational and management-type situations when a quick and clear decision is required
- when 3 report format templates give several people a clear next step

### Do not use this when

- if there is insufficient detail, you cannot cover the risk with human control

### Connection points

- daily handovers between financial analyst and operational manager
- short statuses and decision preparation notes
- repetitive tasks, where 3 different usable reports are required for 3 target groups

## When

### Use it when

- preparing a weekly report is a task from an operative and managerial point of view, and time is short
- Based on 3 report format templates, several actors must act in the same way

### Frequency

At least once per workday, and every time for recurring tasks.

### Late warning signs

- 3 report format templates need to be rewritten every time
- you often get back that the content does not handle the problem of insufficient detail

## Why it matters

### Practical benefits

- 3 report format templates are created faster, with fewer rounds
- quality will be more measurable for 3 target groups with 3 different usable report indicators
- the rework is reduced, because you filter out errors of insufficient detail early on

### What happens if you skip it

- the number of misunderstandings is increasing in the operational and management view of the weekly report
- it is more difficult to maintain 3 different usable report goals for 3 target groups

### Realistic expectations

- It will improve: the practical application of report and analysis templates and the quality of the deliverable output.
- Not guaranteed: infallible response without human control in every run.

## How

### Step by step

1. Name the work situation and the expected result exactly. 2. Write down the mandatory limits (time frame, scope, style, prohibitions). 3. Request the first AI version in a specific format. 4. Run a 3-point quality gate, paying special attention to the risk of **inadequate detail**. 5. Fix the output, then save it as a usable template.

### Do / Don't

**Do**
- Enter a short, specific task description so that the format of the 3 report format templates is clear from the start.
- Before finalizing, check whether 3 different usable report measurement points have been met for 3 target groups.


**Don't**
- Do not leave the topic of insufficient detail in the background, even if the answer seems good at first.
- Don't ask for a general text if you need material for a specific decision. Preparation of a weekly report from an operative and managerial perspective for a task.

### Common errors and their correction

- **Error:** a too general request is made to create a weekly report on the situation in the operational and management view. **Correction:** name the person concerned, the deadline and the decision target.
- **Error:** there is no connection between the output and 3 different usable report measurements for 3 target groups. **Fix:** record 1 binary pass/fail result at the end of each run.

### It's ready if

- [ ] **3 report format template** outputs are ready and saved.
- [ ] You checked the output according to at least 3 quality criteria.
- [ ] You have documented 3 different usable report** indicators for the **3 target groups.

## Guided exercise

### Inputs

- a short description of a real task about the following situation: **Creating a weekly report in an operative and managerial view**
- a short sample of your own that shows what you consider a good report and analysis template output
- checklist to exclude the risk of insufficient detail

### Steps

1. Write in 4 lines the situation, the goal and how you will measure 3 different usable report results for 3 target groups.
2. Create a sample request to generate 3 report format templates, then request two different versions.
3. Compare the versions in the table and mark which one can go on for real use.

### Expected output format

| Element | What do you type | Expected result |
| --- | --- | --- |
| Situation | Preparation of a weekly report in operative and managerial view | Everyone involved understands from one reading |
| Request frame | Target + context + format + constraint | Minimum of 2 usable versions |
| Quality gate | Accuracy + Risk + Next Step | Clear repair decision |
